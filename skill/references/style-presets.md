# 风格预设（Style Presets）

> 风格是场景驱动的，格式是载体驱动的。这两条轴独立组合。
>
> - **风格预设**决定修辞策略、词汇域、语调参数——同一个风格在邮件、短信、Push、弹窗里修辞一致，但格式不同
> - **载体约束**决定字数、结构、标点——同一个载体可以承载不同风格
>
> 使用方式：先根据场景+品牌推断风格预设，再根据载体应用格式约束。agent 有基本判断力——用户说"写一条短信"，agent 应该知道短信没有换行和落款，不需要 skill 逐条规定。

---

## 风格判定逻辑

### 路径 1：用户指定风格

用户明确说"用克制高级的风格""用促销话术""严谨一点"——直接匹配预设。

### 路径 2：场景自动推断

用户未指定风格时，根据输入信息推断：

| 推断信号 | → 风格预设 |
|---------|-----------|
| 品牌/创意/灵感/课程/联名/质感/高级 | 克制高级 |
| 折/省/限时/抢/大促/补贴/低价/秒杀 | 强推促销 |
| 金融/银行/证件/安全/过期/合规/身份/验证 | 严谨合规 |
| 新手/引导/第一次/试试/学习/入门 | 温暖陪伴 |
| B端/企业/效率/方案/工具/团队/协作 | 中性专业 |
| 失败/出错/异常/重试/解决 | 建设性解法（UI 场景预设，见 scene-templates.md） |
| 庆祝/成功/完成/恭喜 | 温暖庆祝 |

**推断后标注"推断为 [风格]，请确认"**——与 SKILL.md 中文案类型推断逻辑一致。

### 冲突处理

当推断信号冲突时（如"金融产品的大促"），**场景优先于品牌**：
- 金融大促 → 严谨合规的底色 + 强推促销的利益表达，但禁用"抢""手慢无"等词
- 创意工具的安全提醒 → 克制高级的语调 + 严谨合规的事实陈述，但不用"灵感""创作"

## 五种风格预设

### 一、克制高级

| 维度 | 参数 |
|------|------|
| FORMAL | 1-2 |
| WARMTH | +1 ~ +2 |
| FORCE | 1-2 |
| **修辞策略** | 禀赋效应、正向框架、低门槛引导 |
| **核心词汇域** | 灵感、创作、值得、助力、陪伴、探索、发现、开启、继续、专属 |
| **禁用词汇** | 限时、仅剩、抢、马上、手慢无、不容错过、赶紧 |
| **感叹号** | 不用 |
| **叠词** | 可用（试试、看看、逛逛） |
| **数字锚定** | 含蓄使用——"6.8折起"可以，"仅剩3件"不行 |

**修辞特征**：
- 用"值得"替代"需要"——"值得有更多积分去实践" > "你需要购买积分"
- 用"继续"替代"开始"——暗示用户已经拥有，降低心理门槛
- 用"专属"替代"优惠"——提升价值感，降低促销感
- 利益点放在"你可以做什么"而非"你省了多少钱"

**跨载体表现**：

| 载体 | 示例 |
|------|------|
| 邮件 | 居中短标题 + 1-2 段正文 + CTA。问候有温度但不粘腻。结尾有品牌语。 |
| Push | "你的专属礼包已就绪" > "限时优惠抢购" |
| SMS | "【XX】你的专属6.8折已到账，3档可选，90天有效。去看看>x.xx/xxx 回T退订" |
| 弹窗 | 标题："专属礼包已就绪" 正文："学员专属6.8折，3档可选" 按钮："查看礼包" |
| Toast | "礼包已到账" |

### 二、强推促销

| 维度 | 参数 |
|------|------|
| FORMAL | 1-2 |
| WARMTH | +0 ~ +1 |
| FORCE | 3-4 |
| **修辞策略** | 损失厌恶、数字锚定、紧迫感、社交验证 |
| **核心词汇域** | 折、省、仅剩、限时、抢、专属价、到手价、立减、倒计时、已拼、人手 |
| **禁用词汇** | （几乎无禁用，但遵守伦理红线——确认shaming禁用，虚假稀缺禁用） |
| **感叹号** | 利益场景可用 1 个，安全/错误场景禁用 |
| **叠词** | 选择性使用——CTA 元素可局部突破 WARMTH 范围（"看看、逛逛"WARMTH+2），标题和正文不用叠词（"抢抢"削弱紧迫感） |
| **数字锚定** | 积极使用——"省30%""3亿人都在用""仅剩3天" |

**修辞特征**：
- 数字前置——"6.8折"放在最前面而非最后
- 紧迫感——时间限定（"仅剩3天"）+ 数量限定（"限量500份"），但必须真实可验证
- 力度来自 FORCE，不是 WARMTH——"限时5折"比"快来看看5折优惠"更有力；温暖度保持在 +0~+1，避免亲昵感削弱紧迫感
- 社交验证——"3人已拼，仅差你1人""已抢10万件"
- CTA 用强动词——"立即抢购""马上领取" > "查看""了解"

**跨载体表现**：

| 载体 | 示例 |
|------|------|
| 邮件 | 利益点前置，紧迫感明确，CTA 强动词。可省略品牌语。 |
| Push | "3折起！仅剩3天" > "你的专属优惠已到账" |
| SMS | "【XX】限时5折，年度最低价！仅剩3天，点击查看>x.xx/xxx 回T退订" |
| 弹窗 | 标题："限时6.8折" 正文："年度最低价，仅剩3天" 按钮："立即抢购" |
| Toast | "优惠即将过期" |

**⚠️ 伦理红线**：
- "仅剩3件"须有真实库存
- "最后一天"不能永远重复
- 退出选项不能含 confirmshaming（"暂不需要" > "我宁愿花钱"）
- TOFU 阶段禁用损失厌恶——用户刚接触产品时，"再不买就晚了"等于强迫推销

### 三、严谨合规

| 维度 | 参数 |
|------|------|
| FORMAL | 4-5 |
| WARMTH | -2 ~ 0 |
| FORCE | 3-4 |
| **修辞策略** | 正向框架、事实陈述、后果说明、具体措施 |
| **核心词汇域** | 信息、更新、继续、保障、确保、根据、规定、身份、验证、安全、措施、保护 |
| **禁用词汇** | 灵感、创作、助力、开心、试试、看看、逛逛、赶紧、抢 |
| **感叹号** | 禁用 |
| **叠词** | 禁用 |
| **数字格式** | 保留精确数字（"99.4%"不简化为"更高了"） |

**修辞特征**：
- 正向框架表达安全——"更新后可继续使用" > "不更新将无法使用"
- 具体措施 > 空洞形容词——"不解锁付不了，每笔支付都会确认" > "很安全"
- 后果说明要精确——"身份证过期后30天内可正常使用，30天后将限制部分功能" > "不更新会影响使用"
- "我们"用于责任归属——"我们将保护您的个人信息" > "您的信息受保护"

**跨载体表现**：

| 载体 | 示例 |
|------|------|
| 邮件 | 完整标点、完整句子、"我们"主语、后果精确说明、管理路径 |
| Push | "身份证即将过期" > "你的身份证快过期啦" |
| SMS | "【XX】您的身份证将于30天后过期，届时部分功能将受限。请及时更新>x.xx/xxx 回T退订" |
| 弹窗 | 标题："身份证即将过期" 正文："您的身份证将于2026年6月20日过期，过期后部分功能将受限。请及时更新以确保正常使用。" 按钮："更新信息" |
| Toast | "身份证即将过期" |

## 风格冲突处理

当场景信号冲突时，**优先级：场景类型 > 品牌调性 > 用户指定**。

| 冲突场景 | 处理方式 | 结果 |
|---------|---------|------|
| 金融产品的大促 | 场景（合规）> 品牌（促销） | 严谨合规底色 + 利益表达，但禁用"抢""手慢无" |
| 创意工具的安全提醒 | 场景（安全）> 品牌（温暖） | 正向框架 + 事实陈述，不用"灵感""创作" |
| B端产品的 Onboarding | 场景（引导）> 品牌（专业） | 温暖陪伴的低门槛 + 中性专业的功能陈述 |
| 教育产品的支付 | 场景（支付）> 品牌（温暖） | 中性专业的精确表述 + 顾虑消除 |

**混合风格的写法**：不是两种风格各取一半，而是**底色用场景风格，修辞用品牌风格的词汇域**。

示例——"金融产品的大促邮件"：
- 底色：严谨合规（FORMAL=4, WARMTH=0, 正向框架, 完整标点）
- 修辞：允许利益数字前置，但不用"抢""手慢无"
- 结果："【XX银行】专属理财，年化4.2%起。限额发售，先到先得。"——数字锚定有，紧迫感有，但措辞克制。

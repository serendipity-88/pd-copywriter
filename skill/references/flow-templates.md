# 多步骤引导流程模板

> 每个模板是跨步骤的文案组装图——步骤架构、信息递进、CTA 升级、导航文案、变体示例。
> 所有示例用占位符（`[产品名]` `[功能]` `[核心利益]`），适用于任何产品。
> 微信、抖音、淘宝等真实产品案例放在"参考案例"中。

## 一、新手引导流程

### 步骤架构

**3-4 步是最佳范围**。5 步是上限，6+ 步可见步骤导致 >50% 流失（OnboardMap）。

| 步骤位置 | 内容类型 | 目的 | CTA 承诺度 |
|---------|---------|------|-----------|
| 第 1 步 | 价值主张/核心利益 | "为什么要用" | 极低：好奇心驱动 |
| 第 2 步（可选） | 身份/偏好选择 | "这是给我的" | 低-中：认同驱动 |
| 第 3 步 | 快速成功/首次体验 | "真的有用" | 中：行动驱动 |
| 最后一步 | 权限/激活 | "我决定用了" | 高：承诺驱动 |

### 信息递进

```
核心利益（为什么）→ 个性化（为谁）→ 快速成功（真的行）→ 承诺激活（我加入）
```

**关键原则**：先"为什么"再"怎么做"。第一步必须回答"我为什么要用"，不是"这个功能怎么用"。

### 必须遵守的规则

1. **第一步 < 90 秒**：用户能在 90 秒内完成第一步，否则流失率急剧上升
2. **权限请求放最后**：先展示价值再请求权限——延迟推送请求可将授权率从 ~25% 提升到 ~50%
3. **CTA 预览下一步**："设置账户" > "继续"——每个按钮告诉用户下一步是什么，不是泛泛的"下一步"
4. **步骤数 ≤ 4**：3-4 步最佳，5 步是上限。超过 5 步必须用渐进式披露（不显示总步骤数）
5. **跳过按钮用"稍后"或"暂不需要"**：不用"跳过"——"跳过"暗示拒绝，"稍后"暗示推迟
6. **进度条从部分填充开始**：禀赋效应——已完成的进度感增加完成意愿
7. **禁止在引导中放设置/法律条款**：设置无可见效果，法律条款破坏引导体验

### 变体示例

**变体 A：3 步标准型**（大多数 C 端产品）

## 二、功能教育卡片

### 结构

| 元素 | 字数 | 说明 |
|------|------|------|
| 提示文案 | 8-20 字 | 一句话说清这个功能做什么 / 用户获得什么 |
| 操作按钮 | 2-5 字 | 确认型（"我知道了"）或行动型（"试试看"） |
| 关闭/跳过 | 2-4 字 | 可选，非阻塞型必须有 |

### 信息递进

```
触发时机 → 一句话价值 → 行动出口
```

**核心原则**：功能教育卡片不是教程，是提示。用户不需要"学会"什么，只需要"知道"有这个功能。

### 必须遵守的规则

1. **即时触发，不阻断**：在用户首次遇到该功能时触发，不在 App 启动时批量展示
2. **单步为主**：中国产品的主流模式是单步半透明遮罩 + 高亮 + 一句话 + "我知道了"——不用多步卡片序列
3. **"我知道了" > "跳过"**：确认型文案（"我知道了""好的""明白了"）优于拒绝型（"跳过""关闭"）——确认是认知接收，拒绝是排斥
4. **一句话说清价值**：不是"点击这里可以XX"，而是"XX帮你YY"——功能名 + 用户利益
5. **允许再次查看**：被关闭的教育卡片必须有途径重新触发（帮助中心、设置、首次触发入口）

### 触发时机分类

| 触发方式 | 说明 | 适用场景 | 示例 |
|---------|------|---------|------|
| 首次使用触发 | 用户第一次进入某功能时 | 功能入口教育 | 首次点击"扫一扫"时提示扫码方式 |
| 条件触发 | 满足特定条件时 | 进阶功能教育 | 余额 > 1000 时提示理财功能 |
| 红点发现 | tab/icon 上的红点标记 | 新功能发现 | 新功能入口标红点，用户主动点击 |
| 空状态引导 | 内容为空时的引导 | 首次使用引导 | "暂无订单，去逛逛" |

### 变体示例

**变体 A：半透明遮罩型**（中国产品主流）

> [一句话功能价值]
> [我知道了] / [试试看]

示例：长按消息可以撤回 → [我知道了]

**变体 B：红点发现型**（非阻塞）

> （红点标记在 icon/tab 上，用户主动点击后展示功能说明）

示例：新功能入口标红点 → 用户点击 → 展示功能卡片

**变体 C：首次使用引导型**（功能入口内）

> [功能名]，[核心利益]
> [操作提示]
> [试试看] / [暂不需要]

示例：[功能名]，解锁就能用 → 碰一下手机背面即可 → [试试看] / [暂不需要]

### 参考案例

| 产品 | 教育方式 | 关键洞察 |
|------|---------|---------|
| 微信 | 红点发现，无教育文案 | 非阻塞，用户自主发现 |
| 抖音 | 首次操作触发 | 即时触发，在用户需要时才教 |
| Figma | 条件触发 | 用户添加文本框时才提示，精准时机 |

## 三、步骤导航文案

### 步骤指示器

| 类型 | 文案 | 适用条件 |
|------|------|---------|
| 数字型 | "1/3""2/3""3/3" | 步骤数 ≤ 4 且步骤固定 |
| 进度条 | 无文案，视觉指示 | 步骤数 > 4 或不想暴露总步骤数 |
| 不显示 | 无 | 2 步流程或即时触发型教育 |

**关键规则**：步骤数 > 5 时不显示总步骤数（"步骤 3" 而非 "步骤 3/8"），避免用户产生畏难情绪。

### 导航按钮文案

| 位置 | 推荐文案 | 避免 | 说明 |
|------|---------|------|------|
| 前进 | [预览下一步的具体动作] | "下一步""继续" | 每个按钮告诉用户下一步做什么 |
| 后退 | "返回" / "上一步" | — | 中性陈述 |
| 跳过 | "稍后" / "暂不需要" / "我自己看看" | "跳过" | 推迟而非拒绝 |
| 完成 | [具体完成动作] / "开始使用" | "完成""确定" | 承诺具体结果，不是抽象确认 |

### CTA 升级规则

| 步骤位置 | CTA 承诺度 | 文案模式 | 示例 |
|---------|-----------|---------|------|
| 第 1 步 | 极低 | 好奇心驱动 | "看看怎么用""了解一下" |
| 中间步骤 | 低-中 | 认同/行动驱动 | "选好了""设置我的" |
| 最后一步 | 高 | 承诺具体结果 | "开始使用""创建我的第一个XX" |

**核心规则**：CTA 从低承诺升级到高承诺。"了解" < "试用" < "创建" < "开始使用"。禁止全程使用"下一步"——每个 CTA 应预览下一步的具体内容。

## 四、渐进式披露规则

### 何时用哪种模式

| 模式 | 适用场景 | 触发方式 | 典型产品 |
|------|---------|---------|---------|
| 零引导 | 核心价值直观可发现（内容流、聊天） | 无——用户直接使用 | 抖音、微信 |
| 兴趣选择 | 需要个性化但功能简单 | 首次启动 | 淘宝、小红书 |
| 分步引导 | 功能复杂、操作路径不直观 | 首次启动或首次使用 | 工具类、B端产品 |
| 即时教育 | 新功能、进阶功能 | 首次触发该功能时 | 通用——所有产品 |
| 红点发现 | 新入口、新 tab | 用户主动点击红点 | 中国产品主流 |

### 披露层级规则

| 规则 | 说明 | 数据支撑 |
|------|------|---------|
| 最多 2 层披露 | 超过 2 层用户会迷失 | NN/g 46 个应用研究 |
| 即时 > 预先 | 在用户需要时才展示教育，不在启动时批量展示 | NN/g：移动端教程不改善任务表现 |
| 权限请求延迟 | 先展示价值再请求权限 | 推送授权率从 ~25% 提升到 ~50% |
| 可关闭 + 可找回 | 用户关闭教育后必须有途径重新查看 | NN/g：不能重新找到帮助的用户默认不使用该功能 |

### "什么时候不用引导"检查清单

| 条件 | 说明 |
|------|------|
| 核心价值在首页即可发现 | 直接让用户用，不展示引导 |
| 功能操作路径直观（< 2 步可完成） | 用空状态引导代替教育卡片 |
| 用户对该类产品已有心智模型 | 不教用户已知操作 |
| 交互范式非全新 | 不需要教程，最多即时提示 |

满足任意 2 条 → 不用引导，最多即时提示。
